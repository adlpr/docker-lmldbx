Content Guide

Introduction

This site was built to showcase the efforts towards automated mapping from Lane Medical Library's MARC catalog data into the XOBIS schema. Functionality is currently limited to single record views and simple queries, but we hope to expand this in the near future to include search and advanced relationship exploration views.

XOBIS (the XML Organic Bibliographic Information Schema) was first conceived and developed in 2001–2002 by the MedLane Project at Lane, inspired by the flexibility of the XML format and its potential for rectifying the limitations and inefficiencies of cataloging in MARC. It provides an entity-relationship model for conceptually unifying bibliographic and authority data, conceiving the universe of library metadata in ten Principal Elements representing fundamental categories of information: Work, Being, Concept, Event, Object, Organization, Language, Place, Time, and String. Each of these elements in turn comprises:

  • an identifying Entry (name and qualifiers);
  • a list of Variant entries (see-from references);
  • authorized and qualified Relationships to other elements;
  • free-form Notes of authorized types;
  • administrative and other Control Data.

Records

The /record/ endpoint displays a single record view, along with any links to attached holdings. The default is an HTML view generated from the XML via XSLT; passing the parameter ?format=xml returns the record in pure XOBIS-XML.

Related Records

The /related/ endpoint returns a paginated list of Records which all contain at least one Relationship with a given target Record; i.e., an inverted list of Relationships for that Record.
